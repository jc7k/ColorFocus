# README Structure

Standard structure for project README files.

## Required Sections (in order)

1. **Title + Tagline** — Project name and one-line description
2. **Live Demo** — Link to deployed app (if applicable)
3. **Why I Built This** — Personal story/motivation behind the project
4. **Features** — Bulleted list of capabilities, grouped by category
5. **Quick Start** — How to run locally (clone, install, run)
6. **Tech Stack** — Languages, frameworks, hosting, key dependencies
7. **Development** — Prerequisites, setup, running tests
8. **Contributing** — How others can contribute
9. **License** — License type with link to LICENSE file
10. **Built with AI** — Credit AI tools used in development

## Optional Sections

- **Supported Languages** — For i18n apps, table of language support
- **Configuration** — Tables for presets, difficulty levels, etc.
- **Deployment** — Platform-specific deploy instructions
- **Future Work** — Planned features and improvements
- **Support** — Donation links, sponsorship info
- **Acknowledgments** — Credits and thanks

## Formatting Rules

- Use tables for structured data (features, config options)
- Include code blocks for all CLI commands
- Add disclaimer if app has medical/legal implications
- Keep sections scannable — bullets over paragraphs
